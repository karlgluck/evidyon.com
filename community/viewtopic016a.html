Yeah I'm sad I can't just use Xen's content and change the game ...but there is a possibility available: I *think* it might be possible to use Xenimus's content if I don't distribute the content itself, ie. require that Xenimus is installed before installing Volucris, and just "snatching" all of the content directly from the Xenimus directory. There are a few problems with this approach, however:
  • If EJ changes any of the models/model formats or file names, it screws up Volucris
  • JUpdates would have to occur every time EJ updates
  • If Volucris got popular, EJ would likely encrypt everything

I might be able to offer a "mirror" download of Xenimus that I just..."forget"...to update that is compatible with Volucris, but still that gets into a legal gray area. It's better to simply have custom content--the problem therein, however, lies in the fact that it is very difficult to create.

I will upload some of EJ's models as .X files so you modelers can load them up and play with them.


Also, to answer Vitriol's original question, no the game won't be called Volucris. That's a "beta" name, like Longhorn was for Windows Vista.
