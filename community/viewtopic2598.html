by Karl G.
First: yes, that's fairly easy to do; however, if you check the timeline, I've moved animated meshes back by three weeks to make way for gameplay (since you can play the game as a cube, I think that's more important than having an animated character that doesn't do anything...)

Second and Third: These customization features are doable. The only real restraint is the amount of time it takes to create the customizations: things like hair color and skin tone aren't difficult to do individually, but once you start combining more and more features, you multiply the number of textures required. For example, a naked human with 1 type requires 1 texture: add three hair colors, that's three textures; add five skin tones, thats 15 textures (one for each hair color/skin tone pair); add tattoos or scars, or facial features...you get the point. Yes, it's possible. It's something to think about for the future. However, my logic here is the same as for pushing animated mesh integration back: it's more useful to have playable, balanced, fun game-play than be able to give your player earrings.

An interesting approach would be to have the game come with only the default textures, but allow players to create their own content and share with other players. The server wouldn't even have to host the player content, the clients would be able to transfer the textures peer-to-peer, saving on bandwidth. Of course, to keep control on what textures are allowed, new textures must be sent to server and approved by the administrators. A hash of the texture can then be calculated, and the server would store a database of approved hashes. clients would need to check with the server before letting the player use a new texture. Once a texture is approved, it does not need to be stored on the server anymore, as the transfer occurs directly from client to client.
